# Standalone GEPA Prompt Optimization

This directory contains the standalone code needed to reproduce the GEPA prompt optimization for LegalLINK-style legal Q&A generation.

The optimizer uses DSPy's `dspy.GEPA` over a single DSPy module that emits:

- `reasoning`
- `question`
- `answer`

The default dataset is `disi-unibo-nlp/eurlex_relations`, with input fields `text1`, `text2`, and `relation_types`.

## 1. Create an environment

Use Python 3.10 or newer. GEPA currently declares support for Python versions below 3.15.

```bash
cd src_prompt_optimization
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
cp .env.example .env
```

If `vllm>=0.18.1` is not available on the target machine/index, install the newest compatible vLLM build for the machine's CUDA/PyTorch stack and keep the rest of the requirements unchanged.

## 2. Deploy an OpenAI-compatible model server

Use the included vLLM deployment script:

```bash
MODEL_NAME=openai/gpt-oss-20b \
GPU_ID=0 \
SERVER_PORT=8000 \
bash deploy_vllm_model.sh
```

The script starts vLLM in the background, writes logs to `src_prompt_optimization/logs/`, records a PID under `src_prompt_optimization/pids/`, and waits until `http://127.0.0.1:8000/v1/models` is reachable.

Stop the server with:

```bash
SERVER_PORT=8000 bash stop_vllm_model.sh
```

Equivalent raw vLLM command:

```bash
CUDA_VISIBLE_DEVICES=0 vllm serve openai/gpt-oss-20b \
  --host 0.0.0.0 \
  --port 8000 \
  --served-model-name openai/gpt-oss-20b \
  --tensor-parallel-size 1 \
  --max-model-len 38000
```

The optimization script can also use two servers: one for generation and one for judging/reflection. In that case, pass different `--task_port` and `--eval_port` values.

## 3. Run a small smoke test

From this directory:

```bash
TRAIN_SAMPLES=8 VAL_SAMPLES=4 MAX_FULL_EVALS=2 NUM_THREADS=2 \
MODEL_NAME=openai/gpt-oss-20b \
bash run_single_vllm_gepa.sh
```

## 4. Run the optimization

```bash
MODEL_NAME=openai/gpt-oss-20b \
TRAIN_SAMPLES=2000 \
VAL_SAMPLES=1000 \
MAX_FULL_EVALS=25 \
NUM_THREADS=32 \
OUTPUT_DIR=results/gepa_optimization_single_model \
bash run_single_vllm_gepa.sh
```

The main script can also be run directly:

```bash
python3 run_gepa_qa_optimization.py \
  --dataset_name disi-unibo-nlp/eurlex_relations \
  --dataset_split train \
  --train_samples 2000 \
  --val_samples 1000 \
  --task_model openai/gpt-oss-20b \
  --eval_model openai/gpt-oss-20b \
  --reflection_model openai/gpt-oss-20b \
  --vllm_url http://127.0.0.1 \
  --task_port 8000 \
  --eval_port 8000 \
  --max_full_evals 25 \
  --output_dir results/gepa_optimization_single_model
```

Exactly one budget argument is required: `--gepa_auto {light,medium,heavy}`, `--max_full_evals N`, or `--max_metric_calls N`.

## Outputs

The output directory contains:

- `optimized_program_*.json`: DSPy serialized optimized program when supported by the installed DSPy version.
- `optimized_program_*.txt`: extracted optimized instructions.
- `optimization_results_*.json`: run metadata and optimized instructions.
- `gepa_optimization_stats.csv`: append-only score summary.
- `generated_qa_*.jsonl`: Q&A generated by the optimized program on the validation sample.

## Notes for replication

- The script uses an OpenAI-compatible endpoint. vLLM does not need a real API key; `OPENAI_API_KEY=EMPTY` is enough.
- GEPA depends heavily on informative metric feedback. The metric in this directory returns both a numeric score and textual feedback for failed constraints.
- For a real run, use at least tens of examples. The smoke-test settings only verify plumbing.
- Keep train and validation samples fixed with `--seed` when comparing prompt variants.
